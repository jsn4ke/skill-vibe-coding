│   │   └── skill-design-decompose/      # 新增：技能设计拆解
│   │       ├── SKILL.md

│   │           └── category-system.md   # 技能分类体系

## 核心工作流

```
输入: 技能名称 或 技能描述
    │
    ▼
┌──────────────────────────────────┐
│ Phase 1: 收集技能信息             │
│                                  │
│  WoW 技能？                      │
│    ├─ 是 → WebSearch 搜 Wowhead  │
│    │       提取：效果、数值、      │
│    │       冷却、施法时间、描述    │
│    └─ 否 → 使用用户提供的描述     │
│           可能追问不明确的细节     │
└──────────┬───────────────────────┘
           │
           ▼
┌──────────────────────────────────┐
│ Phase 2: TC 机制参考              │
│                                  │
│  调用 tc-mechanism-ref 查询：     │
│  - 技能涉及的机制类型             │
│  - TC 中类似技能的实现方式        │
│  - 相关效果类型和 aura 类型       │
└──────────┬───────────────────────┘
           │
           ▼
┌──────────────────────────────────┐
│ Phase 3: 框架对照拆解             │
│                                  │
│  对每个组件：                     │
│  ├─ WoW 有对应机制 → 直接映射     │
│  │  (标注 TC 效果类型/aura 类型)  │
│  └─ WoW 无对应 → 自创机制设计     │
│     (说明设计思路和原因)          │
└──────────┬───────────────────────┘
           │
           ▼
┌──────────────────────────────────┐
│ Phase 4: 输出设计文档             │
│                                  │
│  按标准模板输出到                 │
│  skill-designs/<skill-name>.md   │
└──────────────────────────────────┘
```

## 输出文档结构

```markdown
# <技能名称>

> 来源: <WoW/自创> | 生成日期: <date>

## 概述
一段话描述技能的整体定位和用途。

## 基本信息

| 属性 | 值 | WoW 参考 |
|------|-----|----------|
| 分类 | 伤害/治疗/控制/增益/召唤/移动/其他 | TC SpellEffectName 映射 |
| 施法时间 | 即时/ X 秒 / 引导 X 秒 | SpellCastTimesEntry |
| 冷却 | X 秒 / 分类冷却 | SpellCooldownsEntry |
| 充能 | 最大 X 层，恢复时间 X 秒 | SpellHistory ChargeEntry |
| 资源消耗 | X 点 <资源类型> | SpellPowerEntry |
| 距离 | X 码 | SpellRangeEntry |
| GCD | 触发 / 不触发 / 延迟 X 秒 | GlobalCooldownMgr |

## 目标

| 属性 | 值 | WoW 参考 |
|------|-----|----------|
| 目标类型 | 单体/区域/锥形/链式/轨迹 | SpellImplicitTargetInfo |
| 目标数量 | X 个 / 无限 | MaxAffectedTargets |
| 目标筛选 | 敌方/友方/自身/... | SpellTargetCheckTypes |
| 半径 | X 码（区域技能） | SpellRadiusEntry |

## 效果拆解

### 效果 1: <效果名称>
- **类型**: <效果类型> (TC: SPELL_EFFECT_XXX)
- **数值**: 基础值 X，每级成长 Y，系数 Z
- **目标**: 作用目标
- **WoW 映射**: 直接映射 / 适配修改 / 自创

### 效果 2: <效果名称>
...

## Aura（如有）

### Aura 1: <Aura 名称>
- **类型**: <Aura 类型> (TC: SPELL_AURA_XXX)
- **持续时间**: X 秒 / 永久 / 直到取消
- **堆叠规则**: 不可堆叠 / 最大 X 层 / 刷新持续时间
- **触发条件**: 无 / Proc 条件
- **周期效果**: 每 X 秒触发一次 (伤害/治疗/能量)
- **WoW 映射**: 直接映射 / 适配修改 / 自创

## WoW 框架映射汇总

| 组件 | WoW 机制 | 映射方式 |
|------|----------|----------|
| 效果1 | SPELL_EFFECT_SCHOOL_DAMAGE | 直接映射 |
| 效果2 | ... | ... |
| Aura1 | SPELL_AURA_PERIODIC_DAMAGE | 直接映射 |
| 自创机制1 | 无 | 新设计 |

## 实现建议

### 整体设计思路
详细描述技能实现的整体架构思路——
数据如何从定义流向执行，各个系统如何协作，
状态如何在施法生命周期中流转。

### 数据配置
- 哪些参数应该是数据驱动（配置表/JSON）
- 哪些需要脚本扩展（无法数据表达的逻辑）
- 配置结构的组织方式

### 施法流程
- 从施法触发到效果生效的完整流程
- 每个阶段的处理逻辑
- 中间可能的分支（暴击、命中判定、免疫等）

### 效果执行逻辑
- 每个效果的执行顺序和依赖关系
- 数值计算公式和修正器链
- 效果间的交互（一个效果的结果影响另一个）

### Aura 生命周期（如有）
- 创建/应用/刷新/移除的触发条件
- 周期效果的计时和执行
- 堆叠变化时的处理
- 被驱散/打断时的清理

### 特殊逻辑（如有）
- 自创机制的设计思路
- 为什么 WoW 框架无法覆盖
- 替代方案的设计理由
```

## 技能分类体系

主分类（互相不排斥，一个技能可属多个）：

| 分类 | 说明 | TC 效果示例 |
|------|------|-------------|
| 伤害 | 直接或间接造成生命值损失 | SCHOOL_DAMAGE, WeaponDamage |
| 治疗 | 恢复生命值 | HEAL, HEAL_PCT |
| 控制 | 限制目标行动能力 | STUN, ROOT, FEAR, CONFUSE |
| 增益 | 强化自身或友方 | MOD_STAT, MOD_SPEED, MOD_POWER |
| 减益 | 削弱敌方 | MOD_RESISTANCE, MOD_DECREASE_SPEED |
| 召唤 | 创建单位或物体 | SUMMON, SUMMON_PET |
| 移动 | 改变位置 | TELEPORT, CHARGE, LEAP, KNOCK_BACK |
| 资源 | 改变资源值 | ENERGIZE, POWER_DRAIN |
| 驱散 | 移除效果 | DISPEL, DISPEL_MECHANIC |
| 触发 | 触发其他技能/效果 | TRIGGER_SPELL, DUMMY |

## Wowhead 搜索策略

搜索关键词构造：
- 中文技能名：`site:wowhead.com <技能英文名>`
- 直接搜 Wowhead 数据库：`<技能名> wowhead`
- 优先查看 spell 页面的 Effects、Tooltips、战报数据
- 关注：效果列表、数值、持续时间、冷却、施法时间、范围

## 约束

- **输出文档供讨论**，不生成代码
- **优先 WoW 框架**，无法映射再自创
- **实现建议要详细**，涵盖逻辑和设计思路，但不需要代码
- **自创机制需说明原因**，为什么 WoW 框架不够用
